# 2.0 Preview 001 Release Notes

## 1. Release identity

- Release label: `2.0-preview-001`
- npm package version: `2.0.0-preview.1`
- Maintainer-only tag (NOT cut by agents): `v2.0.0-preview.1`
- Status: `preview вЂ” implementation Tasks 1-8 complete; validation pending Tasks 10-11`

## 2. What's new (high level)

- D+E hybrid memory architecture with optional F through explicit `@graphiti` markers.
- Deterministic capture across `chat.message`, `tool.execute.after`, `message.part.updated`, `session.idle`, and `session.compacted`.
- Shadow extractor enabled by default, with schema validation, timeout isolation, concurrency limits, and fail-open behavior.
- Explicit user markers using `@graphiti Class: body` syntax.
- Bounded top-N recall replaces first-turn broadcast injection by default.
- Existing `graphiti` MCP tool now supports `mode: "recall"` with optional `query` and `topN` arguments.
- Single sanitizer boundary applies to deterministic, shadow, marker, compaction, and migration writes.
- Migration CLI maps legacy `[TYPE: x]` episodes into the typed memory taxonomy.

## 3. Memory taxonomy

### Deterministic classes

- `UserInstruction`: User instructions stored chronologically with session and message metadata.
- `Restriction`: User or project rules, bans, and constraints.
- `StylePreference`: Writing, coding, and interaction preferences.
- `Problem`: Errors, failing checks, symptoms, and active issues.
- `FixAttempt`: Attempts to solve a problem, including failed or expected-success attempts.
- `Achievement`: Verified outcomes, successes, and completed work.
- `FileEdit`: File-change metadata captured from edit-like tool events.
- `CommandRun`: Shell or command metadata captured from command-like tool events.

### Shadow classes

- `ArchitecturalDecision`: ADR-style technical decisions with context and consequences.
- `Decision`: Smaller tactical decisions worth keeping for later.
- `BusinessEntity`: Domain actors, objects, or concepts.
- `BusinessProcess`: Domain workflows and process rules.
- `UseCase`: User goals, scenarios, or acceptance-level behavior.
- `InfrastructureComponent`: Deployment, service, storage, or runtime components.
- `DataModel`: Data structures, schemas, records, and relationships.
- `Strategy`: Plans, approaches, and longer-running technical strategies.
- `Reflection`: Learned patterns, retrospective insights, and reusable lessons.

## 4. Configuration

Configuration precedence is `defaults -> graphiti.jsonc (global / project) -> environment variables`. The global and project `graphiti.jsonc` files merge section by section, then supported environment variables override the merged result.

Default memory sections from `src/config.ts`:

```jsonc
{
  "memory": {
    "enabled": true
  },
  "capture": {
    "enabled": true,
    "trivialMessageMinLength": 4,
    "explicitClassMarkers": ["@graphiti"],
    "ratificationKeywords": {
      "positive": ["works", "good", "thanks", "perfect", "merged", "great"],
      "negative": ["wrong", "no", "doesn't work", "revert", "broken"]
    },
    "ratificationWindowTurns": 1,
    "unverifiedAutoExpireMs": 86400000
  },
  "shadowExtractor": {
    "enabled": true,
    "timeoutMs": 8000,
    "maxConcurrency": 1
  },
  "recall": {
    "enabled": true,
    "topN": 5,
    "broadcastCompat": false
  },
  "markers": {
    "enabled": true,
    "prefix": "@graphiti"
  }
}
```

Supported environment variables:

- `GRAPHITI_MEMORY_ENABLED`: Enables or disables all preview memory features.
- `GRAPHITI_CAPTURE_ENABLED`: Enables or disables deterministic capture.
- `GRAPHITI_SHADOW_ENABLED`: Enables or disables shadow extraction.
- `GRAPHITI_SHADOW_TIMEOUT_MS`: Overrides `shadowExtractor.timeoutMs`.
- `GRAPHITI_SHADOW_PROVIDER`: Selects the configured shadow provider name for operators that wire a provider.
- `GRAPHITI_SHADOW_MODEL`: Selects the configured shadow model name for operators that wire a provider.
- `GRAPHITI_RECALL_TOP_N`: Overrides `recall.topN`.
- `GRAPHITI_RECALL_BROADCAST_COMPAT`: Enables or disables legacy broadcast-compatible recall fallback.

## 5. Capture behaviour

`chat.message` capture ignores trivial messages, very short messages, code-fence-only messages, and emoji-only messages. Explicit marker lines are parsed first and written with `source: "marker"`; marker lines do not prevent the remaining user text from being considered for deterministic capture.

For user messages, regex detectors classify the text as `Restriction`, `StylePreference`, `Problem`, `FixAttempt`, or `Achievement` when matching patterns are present. After pattern detection, the full user message is written chronologically as `UserInstruction` with `chronological: true` metadata.

`tool.execute.after` writes deterministic tool facts. Edit-like tools write `FileEdit` records with detected paths. Shell-like tools write `CommandRun` records with command text and optional exit code. `message.part.updated` and `session.idle` are implemented as no-op capture stubs for Task 10 integration proof. `session.compacted` writes an `Achievement` when a summary is present.

## 6. Shadow extractor (preview default = enabled)

The shadow extractor surface is in [`src/services/shadow-extractor.ts`](../../src/services/shadow-extractor.ts). Providers implement `extract({ prompt, signal })` and return a JSON string. The output contract is schema-validated as:

```json
{
  "candidates": [
    {
      "memoryClass": "Decision",
      "name": "short name",
      "body": "memory body",
      "evidence": "supporting quote",
      "confidence": 0.8
    }
  ]
}
```

Only `memoryClass` values from `SHADOW_MEMORY_CLASSES` are accepted. Attempts to emit deterministic classes are rejected as `invalid-output` by schema validation.

Operational behavior:

- Timeout is `shadowExtractor.timeoutMs`, default `8000` ms.
- Provider calls receive an `AbortSignal`; timeout aborts the provider call.
- Concurrency is capped by `shadowExtractor.maxConcurrency`, default `1`.
- Timeout, provider error, invalid JSON, or invalid schema returns no Graphiti write, logs a warning, and leaves deterministic capture plus chat flow unaffected.
- Disable with `shadowExtractor.enabled = false` in config or `GRAPHITI_SHADOW_ENABLED=false` in the environment.

## 7. Explicit `@graphiti` markers

Marker grammar is one marker per logical line:

```text
@graphiti <Class>: <body>
```

The class name is case-insensitive and must resolve to `ALL_MEMORY_CLASSES`. Multi-line continuation uses a trailing backslash:

```text
@graphiti Decision: Use top-N recall for preview \
because broad broadcast injection is no longer the default.
```

Markers inside fenced code blocks are ignored. Inline code spans are removed before marker scanning, so `` `@graphiti Decision: ignored` `` does not create a marker. Marker bodies are raw parser output until write time. The sanitizer still applies before Graphiti storage.

## 8. Top-N recall (replaces broadcast injection)

Recall triggers:

- Session start, implemented as the first user message in a session.
- Post-compaction refresh, cached for one-shot injection on the next chat message.
- Explicit MCP invocation through the existing `graphiti` tool with `mode: "recall"`.

Recall is bounded by `recall.topN`, default `5`. It reads nodes and facts for query-based recall, or recent episodes for an empty first-message query. Results are deduplicated, bounded, and injected as a synthetic `[MEMORY RECALL - top N]` text part only when items are present.

Fail-open behavior: if Graphiti is unavailable or all reads fail, recall returns an empty list, logs a warning, and the session continues normally. Returned text is privacy-stripped with `stripPrivateContent`; fully private items are dropped.

Default behavior does not broadcast all stored memories. Roll back only if needed by setting `recall.broadcastCompat=true`; this is not recommended and remains false by preview default.

## 9. Privacy / sanitizer

The single sanitizer boundary is [`src/services/sanitizer.ts`](../../src/services/sanitizer.ts). Every Graphiti write path passes through it, including deterministic capture, shadow extraction, explicit markers, compaction, and migration.

Redaction markers used by the sanitizer:

- `[REDACTED:api_key]`
- `[REDACTED:token]`
- `[REDACTED:password]`
- `[REDACTED:email]`
- `[REDACTED:env_secret]`
- `[REDACTED:credential]`
- `[REDACTED:fully-private]`

`<private>...</private>` content is stripped before redaction rules run. If the remaining body is empty or fully private, the stored body becomes `[REDACTED:fully-private]`.

## 10. Migration of legacy `[TYPE: x]` episodes

Migration mapping from `LEGACY_TYPE_TO_NEW_CLASS`:

- `project-config` -> `UserInstruction`
- `architecture` -> `ArchitecturalDecision`
- `error-solution` -> `FixAttempt`
- `preference` -> `StylePreference`
- `learned-pattern` -> `Reflection`
- `conversation` -> unmapped, reported and not written.

Default mode is dry-run. Apply requires explicit `--apply`. Source episodes are not deleted.

Commands:

```sh
opencode-graphiti migrate
opencode-graphiti migrate --apply
opencode-graphiti migrate --apply --group-id <id>
opencode-graphiti migrate --limit <n>
opencode-graphiti migrate --help
```

Actual help text:

```text
opencode-graphiti migrate [--dry-run|--apply] [--group-id <id>] [--limit <n>] [--help]

Options:
  --dry-run       Report what would be migrated without writing (default)
  --apply         Write migrated records
  --group-id <id> Target a specific Graphiti group
  --limit <n>     Cap how many episodes are scanned
  --help          Show this help
```

Idempotency: re-running `--apply` writes 0 new records once each source is migrated. Migration detects existing records by `metadata.migration.sourceEpisodeId`, `metadata.migration.source`, or a previously migrated source id. New records carry `metadata.migration.sourceEpisodeId === <source id>`, `metadata.migration.source`, `metadata.migration.migratedAt`, and `metadata.mappedClass`.

## 11. Rollback / disable flags

Config flags:

- `memory.enabled = false`: Disable all memory features.
- `capture.enabled = false`: Disable deterministic capture.
- `shadowExtractor.enabled = false`: Disable shadow extraction.
- `recall.enabled = false`: Disable recall.
- `recall.broadcastCompat = true`: Restore legacy broadcast-compatible recall fallback. This is not recommended.

Environment examples:

```powershell
$env:GRAPHITI_MEMORY_ENABLED="false"
$env:GRAPHITI_CAPTURE_ENABLED="false"
$env:GRAPHITI_SHADOW_ENABLED="false"
$env:GRAPHITI_RECALL_TOP_N="5"
$env:GRAPHITI_RECALL_BROADCAST_COMPAT="true"
$env:GRAPHITI_SHADOW_TIMEOUT_MS="8000"
```

```sh
GRAPHITI_MEMORY_ENABLED=false
```

The stable CI release workflow at [`.github/workflows/release.yml`](../../.github/workflows/release.yml) runs `bun run typecheck`, `bun test`, and `bun run build` on `v*` tag push before publishing to the default npm dist-tag. Its release job is guarded with `!contains(github.ref_name, '-preview')`, so preview-shaped tags cannot publish as stable.

The preview CI release workflow at [`.github/workflows/release-preview.yml`](../../.github/workflows/release-preview.yml) runs from the `preview` branch, preview-shaped tags such as `v2.0.0-preview` or `v2.0.0-preview.1`, and manual `workflow_dispatch`. It resolves the package version as follows:

- Push to `preview`: uses `package.json` when it already contains `-preview`, otherwise appends `-preview.<run_number>`.
- Preview tag push: strips leading `v` and uses the preview version from the tag as-is.
- Manual dispatch with `base_version=2.0.0`: publishes `2.0.0-preview.<run_number>`.

A plain stable tag such as `v2.0.0` remains owned by the stable release workflow. For a preview build from a stable base version, use manual dispatch with `base_version=2.0.0` or push a preview-shaped tag. Preview publishes use `npm publish --tag preview`, not the default stable dist-tag.
